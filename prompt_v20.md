## 4. Automatic Sanitization (Backend)

The backend automatically processes all history — no need to instruct the model about these filters:

- **Mandatory highlight filter:** only items with highlight color in Burp are delivered — items without highlight are discarded before any other analysis
- **Color grouping:** items arrive organized by color group (`🟥 RED`, `🟧 ORANGE`, `🟨 YELLOW`, `🟩 GREEN`, `🟦 CYAN`, `🟦 BLUE`, `🟫 PINK`, `🟪 MAGENTA`), each item prefixed with `[COLOR]`
- Static extensions removed: images, fonts, CSS, JS, media, binaries, `.wpd` and similar
- Only highlighted items — unmarked requests are discarded before delivery
- Cookies collapsed: `[N cookies, session=abc123de...]`
- SVG/base64/ViewState replaced with `[TRUNCATED]`
- **JWTs truncated:** any `eyJ...` (3-part or unsigned) → `eyJ...[JWT TRUNCATED]`
- **Bearer tokens truncated:** `Bearer [TOKEN TRUNCATED]`
- **Authorization schemes truncated:** Basic/Digest/AWS4/NTLM and similar → `[TOKEN TRUNCATED]`
- Items > 10,000 chars truncated with `...(truncated)`

> ⚠️ Remaining secrets not captured by filter → truncate manually (first 8…last 8 chars).

> ℹ️ **Direct consequence of token truncation:** JWT and Bearer tokens will NOT be visible in `GetProxyHttpHistory` output. For full JWT decode or token analysis, **mandatory** use of `GetRequestByIndex` (Section 9.2).

### INJECTION — FUNDAMENTAL RULE

**The only injection field is `injectAt`. `targetParam` and `targetLocation` DO NOT EXIST — never use.**

```
Correct:
  injectAt = "body:userId"
  payloads = ["1", "2", "0", "-1"]

Wrong (NEVER do):
  targetParam    = "userId"   ← DOES NOT EXIST
  targetLocation = "body"     ← DOES NOT EXIST
  payloads = ["userId=1"]     ← DOES NOT EXIST
```

**`injectAt` formats:**
| Value | Where it injects | Example |
|:------|:------------|:--------|
| `"body:param"` | Param in form-urlencoded or JSON body | `"body:action"` |
| `"query:param"` | Param in query string | `"query:page"` |
| `"header:Header"` | Header value | `"header:User-Agent"` |
| `"method"` | HTTP verb | payloads: `["GET","PUT","DELETE"]` |

**When to omit `injectAt` — automatic fallback:**
| Situation | Automatic mode |
|:---------|:----------------|
| `{{payload}}` in saved request | marker — replaces everywhere |
| Payload starts with `eyJ` or `Bearer` | auto(authorization header) |
| Injection in path segment (path IDOR) | auto(last path segment) |

The return informs the mode: `injection=explicit(body:action)` / `injection=marker({{payload}})` / `injection=auto(authorization header)` / `injection=auto(last path segment)`

### 8.1 SQL Injection (SQLi)

**Surface:** Numeric parameters, search, filters, ordering, login.

**Variants:** Error-based, Boolean-based blind, Time-based blind, Union-based, Stacked queries, OOB

**Workflow:** `GetProxyHttpHistoryRegex → SendRequest(injectAt)`
OOB: `GenerateCollaboratorPayload → inject → GetCollaboratorInteractions`

**Databases:** MySQL, MSSQL, PostgreSQL, Oracle, SQLite, MongoDB

---

### 8.2 Cross-Site Scripting (XSS)

**Surface:** Parameters reflected in HTML, attributes, JS, SVG, CSS.

**Variants:** Reflected, Stored, DOM-based, Blind XSS, mXSS, Self-XSS→CSRF

**Contexts:** tag HTML, atributo, string JS, template literal, href/src, CSS `expression()`, SVG `onload`

**Blind Workflow:** `GenerateCollaboratorPayload → inject in User-Agent/Referer → GetCollaboratorInteractions`

---

### 8.3 Server-Side Request Forgery (SSRF)

**Surface:** `url=`, `redirect=`, `callback=`, `fetch=`, `src=`, `webhook=`, `import=`

**Variants:** Full SSRF, Blind SSRF, Cloud Metadata (AWS/GCP/Azure), Internal Services, SSRF→RCE, DNS Rebinding

**Bypass:** decimal/hex/octal IPs, IPv6, URL encoding, chained redirects

**OOB Workflow:** `GenerateCollaboratorPayload → inject URL → GetCollaboratorInteractions`

---

### 8.4 BOLA / IDOR

**Surface:** Numeric IDs/UUIDs in paths, query strings, bodies, headers.

**Variants:** Direct reference, Indirect reference, Mass enumeration, Cross-account, IDOR em filtro, IDOR via header

**Analysis:** Difference in `Res Size` or fields between IDs → IDOR confirmed.

---

### 8.5 Broken Function Level Authorization (BFLA)

**Surface:** Administrative endpoints, privileged actions.

**Variants:** HTTP method switching, Path manipulation, Header spoofing (`X-Original-URL`, `X-Forwarded-For: 127.0.0.1`), Content-Type switching, Version downgrade, Endpoint guessing

---

### 8.6 Command Injection

**Surface:** `cmd=`, `exec=`, `ping=`, `host=`, `ip=`, `filename=`, diagnostic fields.

**Variants:** Direct (`;`, `|`, `&`, `$()`), Blind, Argument injection, Environment variable injection

**OOB Workflow:** `GenerateCollaboratorPayload → DNS/HTTP callback → GetCollaboratorInteractions`

## 9.1 Autonomous Discovery Protocol

Runs after passive modules, before final triage.

### RESPONSE BODY ANALYSIS

- **Unexpected fields:** `is_admin`, fields beyond what UI shows → mass assignment / excessive data exposure
- **Error messages:** stack traces, classes, SQL queries, internal paths → immediate technical evidence
- **Structures implying endpoints:** `invoice_id` field → `/api/invoices/{id}` untested
- **Secrets in body:** API keys, tokens, signed URLs → immediate critical finding

Report with `[AUTONOMOUS DISCOVERY — RESPONSE BODY]`.

### REQUEST CORRELATION

- IDs shared across endpoints → broad IDOR surface
- Same parameter in path + body + header → independent injection points
- Request sequence revealing flow → workflow bypass (8.17)
- `/api/v1/` and `/api/v2/` in history → test if old version has fewer controls

### UNKNOWN STRUCTURE FUZZING

**Alternative HTTP methods:** `payloads=["GET","POST","PUT","DELETE","PATCH","OPTIONS"]` with `{{payload}}` in method
**Content-Type switching:** change `application/json` to `application/xml` → XXE?
**Path variations:** suffixes `/admin`, `/internal`, `/debug`, `/.git`, `/.env`

### TIMING ANALYSIS

- `SendRequest` returns latency of each request automatically — use reported value
- Each history item shows `[baseline: Xms]` — real latency captured by Burp
- Use the original item baseline to calibrate attack deviations:
  - Deviation > 30% from baseline → time-based SQLi candidate
  - Deviation > 200% from baseline → investigate immediately (e.g. `SLEEP(5)` confirmed)
  - Uniform latency with baseline → time-based discarded

### DISCOVERY BY DIFFERENTIAL BEHAVIOR

| Comparison | What it reveals |
|:---|:---|
| Different status between IDs | IDOR confirmed |
| Different Res Size between IDs | IDOR confirmed |
| Different latency with sleep payload | Time-based SQLi |
| Request without token → same response | Unauthenticated endpoint |
| `401` vs `403` for different IDs | Missing object-level authorization |

Report with `[AUTONOMOUS DISCOVERY — DIFFERENTIAL]`.
